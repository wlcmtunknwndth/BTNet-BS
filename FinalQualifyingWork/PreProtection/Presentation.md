## Актуальность

- Опционы используются для хеджирования, спекулятивных стратегий и управления портфелями
- Американский пут-опцион можно исполнить до даты экспирации
- Задача оценки становится задачей **оптимальной остановки**
- Для американского пута нет простой замкнутой аналитической формулы
- Требуются быстрые, воспроизводимые и интерпретируемые численные методы

---

## Цель и вклад

**Цель:** реализовать и верифицировать BTNet для оценки американских пут-опционов.

**Собственный вклад:**

1. Реализован пакет `btnn_bs` на Python/PyTorch
2. Реализованы модели `BTNetEuropean` и `BTNetAmerican`
3. Проведена верификация относительно Black-Scholes и QuantLib CRR(500)
4. Исследован перенос весов European → American
5. Проверена возможность вычисления чувствительностей через autograd

---

## Идея BTNet

BTNet представляет биномиальное дерево CRR как нейросетевую архитектуру.

| Компонент CRR | Компонент BTNet |
|---|---|
| Терминальные выплаты | начальный слой |
| Дисконтированное ожидание | линейный фильтр `W` |
| Обратная индукция | последовательность слоев |
| Раннее исполнение | `max(continuation, exercise)` |

Для американского опциона выбор между ожиданием и исполнением реализуется через **maxout**.

---

## Архитектура моделей

| | `BTNetEuropean` | `BTNetAmerican` |
|---|---|---|
| Назначение | европейский пут | американский пут |
| Основная операция | CRR-индукция | CRR + раннее исполнение |
| Слои | Dense + Conv | Dense + Maxout |
| Интерпретация | без досрочного исполнения | оптимальная остановка |

**Параметры базового эксперимента:**  
`S0 = 0.5`, `T = 1`, `r = 0.05`, `sigma = 0.25`, `n = 9`

## Интерпретация результатов

- Аналитическая инициализация сохраняет смысл риск-нейтральных CRR-весов
- Перенос весов может локально улучшить цену, но не является устойчивым методом
- При высокой волатильности transferred W существенно ухудшает метрики
- Компактное дерево `n = 9` дает близкое приближение к CRR(500)
- Увеличение `n` должно снижать ошибку цены, но увеличивает размер сети

---

## Чувствительности и ограничение

- PyTorch autograd позволяет вычислять Delta, Vega и Theta без ручного вывода формул
- Для американского опциона результаты проверялись конечными разностями
- Ограничение: ReLU/maxout являются кусочно-линейными операциями
- Поэтому Gamma в текущей архитектуре равна нулю почти всюду
- Это не ошибка реализации, а направление для дальнейшей модификации модели
